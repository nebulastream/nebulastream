# Requirements: Tokio-Based Rust Sources

**Defined:** 2026-03-14
**Core Value:** Source authors can write `async fn run(ctx)` with just `allocate_buffer()` and `emit(buffer).await` — all complexity hidden by the framework.

## v1 Requirements

### FFI Foundation

- [x] **FFI-01**: Safe Rust wrapper around TupleBuffer with retain/release in Clone/Drop
- [x] **FFI-02**: Safe Rust wrapper around AbstractBufferProvider for buffer allocation
- [x] **FFI-03**: catch_unwind on all FFI-exposed Rust functions to prevent panic-abort across boundary
- [x] **FFI-04**: cxx bridge declarations for source-related C++ types (TupleBuffer, BufferProvider, BackpressureListener)

### Async Source Framework

- [x] **FWK-01**: Shared Tokio runtime configured via WorkerConfiguration (thread count)
- [x] **FWK-02**: `AsyncSource` trait that handles backpressure, semaphore, and channel internally
- [x] **FWK-03**: `SourceContext` type exposing only `allocate_buffer()` and `emit(buffer).await` to source authors
- [x] **FWK-04**: Per-source Tokio Semaphore for inflight buffer tracking (capacity from defaultMaxInflightBuffers)
- [x] **FWK-05**: C++ BackpressureListener translated to Rust awaitable future
- [x] **FWK-06**: CancellationToken per source for cooperative graceful shutdown
- [x] **FWK-07**: Error surfacing via JoinHandle — panics/errors logged, affected source shut down

### Bridge & Integration

- [x] **BRG-01**: Single bridge thread receiving from async_channel with blocking recv, pushing to Folly MPMC task queue
- [x] **BRG-02**: Channel message carries (buffer, semaphore_permit) so bridge constructs OnComplete callback for permit release
- [x] **BRG-03**: Bounded async_channel with capacity tied to defaultMaxInflightBuffers
- [x] **BRG-04**: SourceHandle variant wrapping async Tokio sources, plugging into RunningSource unchanged

### Validation

- [x] **VAL-01**: Generator test source using async sleep loop that emits incrementing values
- [x] **VAL-02**: End-to-end integration test: Rust source → bridge → task queue → pipeline execution

## v2 Requirements

### Observability

- **OBS-01**: tracing instrumentation on source run() body and emit path
- **OBS-02**: tokio-metrics TaskMonitor export for runtime health monitoring

### Resilience

- **RES-01**: Source-level reconnect/retry helper using backoff crate

## Out of Scope

| Feature | Reason |
|---------|--------|
| Async sinks | Completely different ownership model; doubles scope |
| Replacing Folly MPMC queue | Would require rewriting WorkEmitter, QueryEngine, downstream consumers |
| Per-source Tokio runtimes | Defeats shared scheduling; complicates shutdown ordering |
| Unbounded async_channel | Hides backpressure failures; allows OOM |
| Dynamic source hot-plug at runtime | Requires thread-safe runtime mutation and lifetime coordination |
| Panic recovery / auto-restart | Panics indicate bugs; silent restart hides problems |
| Rewriting existing C++ sources in Rust | This adds a parallel path, not a replacement |
