       ```rust
       pub struct BufferProviderHandle {
           // Raw pointer because cxx SharedPtr<AbstractBufferProvider> may not work
           // for abstract types. The pointer is valid for the lifetime of the source.
           ptr: *mut nes_source_bindings::ffi::AbstractBufferProvider,
       }

       // SAFETY: BufferManager (concrete provider) uses folly::MPMCQueue + atomics
       // for thread-safe buffer allocation. Confirmed in RESEARCH.md.
       unsafe impl Send for BufferProviderHandle {}

       Wait -- AbstractBufferProvider is declared as an opaque cxx type. We can receive it via `Pin<&mut AbstractBufferProvider>` in FFI calls. But SourceContext needs to OWN a reference to it.

       The actual approach: BufferProviderHandle stores a raw pointer received from C++ (the C++ side holds the shared_ptr). Create a C++ bridge function `createBufferProviderHandle(AbstractBufferProvider&) -> *mut AbstractBufferProvider` that returns a raw pointer. OR: use the cxx `SharedPtr` type if cxx can bridge `shared_ptr<AbstractBufferProvider>`.

       Given the cxx limitation with abstract types, use the simpler approach: store a raw `*mut ffi::AbstractBufferProvider` and add an FFI function that passes the provider pointer. The C++ side maintains ownership via shared_ptr; the Rust side borrows via raw pointer for the source's lifetime.

       ```rust
       impl BufferProviderHandle {
           /// SAFETY: ptr must be valid for the lifetime of this handle.
           /// The caller (C++ SourceHandle) must ensure the AbstractBufferProvider
           /// outlives all Rust sources using it.
           pub unsafe fn from_raw(ptr: *mut ffi::AbstractBufferProvider) -> Self {
               Self { ptr }
           }

           pub fn get_buffer_blocking(&self) -> TupleBufferHandle {
               // SAFETY: ptr is valid per from_raw contract
               let provider_pin = unsafe { Pin::new_unchecked(&mut *self.ptr) };
               let buf = ffi::getBufferBlocking(provider_pin);
               TupleBufferHandle::new(buf)
           }

           pub fn buffer_size(&self) -> u64 {
               let provider_ref = unsafe { &*self.ptr };
               ffi::getBufferSize(provider_ref)
           }
       }
       ```

       Note: `BufferProviderHandle` does NOT implement `Clone` -- there is one provider per source. It also does NOT implement `Drop` with any cleanup -- the C++ side owns the provider's lifetime.

    1. **runtime.rs**:
       ```rust
       use std::sync::OnceLock;
       use std::sync::atomic::{AtomicU32, Ordering};
       use tokio::runtime::Runtime;
       use tracing::warn;

       static SOURCE_RUNTIME: OnceLock<Runtime> = OnceLock::new();
       static INIT_COUNT: AtomicU32 = AtomicU32::new(0);

       /// Initialize the shared source Tokio runtime.
       ///
       /// Returns `Ok(())` on success. The `Result` return type is required by the
       /// cxx bridge so that any panic during runtime creation is caught by cxx and
       /// converted to a C++ exception instead of aborting the process (FFI-03).
       ///
       /// If called more than once, subsequent calls are no-ops that log a warning.
       pub fn init_source_runtime(thread_count: u32) -> Result<(), Box<dyn std::error::Error>> {
           let call_number = INIT_COUNT.fetch_add(1, Ordering::SeqCst);
           SOURCE_RUNTIME.get_or_init(|| {
               tokio::runtime::Builder::new_multi_thread()
                   .worker_threads(thread_count as usize)
                   .thread_name("nes-source-rt")
                   .enable_all()
                   .build()
                   .expect("Failed to create source Tokio runtime")
           });
           if call_number > 0 {
               warn!(
                   requested_threads = thread_count,
                   "Source runtime already initialized, ignoring duplicate init call"
               );
           }
           Ok(())
       }

       pub fn source_runtime() -> &'static Runtime {
           SOURCE_RUNTIME.get().expect("Source runtime not initialized. Call init_source_runtime() first.")
       }
       ```

       Note: The function signature `fn init_source_runtime(thread_count: u32) -> Result<()>` in the cxx bridge maps to Rust's `Result<(), Box<dyn std::error::Error>>` (cxx's convention for Result). If the runtime builder's `.expect()` panics, cxx catches the panic and converts it to a C++ exception. The explicit `Ok(())` return handles the non-panic success path.

    Implement `context.rs`:

    ```rust
    use crate::buffer::{BufferProviderHandle, TupleBufferHandle};
    use crate::error::{SourceError, SourceResult};
    use async_channel::Sender;
    use tokio_util::sync::CancellationToken;

    pub struct SourceContext {
        buffer_provider: BufferProviderHandle,
        sender: Sender<TupleBufferHandle>,
        cancellation_token: CancellationToken,
    }

    // SAFETY: SourceContext is Send because:
    // - BufferProviderHandle is Send (unsafe impl, validated: BufferManager is thread-safe)
    // - Sender<TupleBufferHandle> is Send (async_channel guarantee)
    // - CancellationToken is Send (tokio_util guarantee)
    // SourceContext is NOT automatically Send because BufferProviderHandle contains a raw pointer.
    unsafe impl Send for SourceContext {}

    impl SourceContext {
        pub fn new(
            buffer_provider: BufferProviderHandle,
            sender: Sender<TupleBufferHandle>,
            cancellation_token: CancellationToken,
        ) -> Self {
            Self {
                buffer_provider,
                sender,
                cancellation_token,
            }
        }

        /// Allocate a buffer from the C++ BufferProvider.
        ///
        /// This uses `spawn_blocking` because the C++ `getBufferBlocking()`
        /// can block indefinitely waiting for a buffer to be recycled.
        /// Without spawn_blocking, this would starve Tokio worker threads.
        pub async fn allocate_buffer(&self) -> TupleBufferHandle {
            // SAFETY: BufferProviderHandle::get_buffer_blocking is safe to call
            // from any thread (BufferManager is thread-safe).
            // We need to send the raw pointer to the blocking thread.
            let ptr = self.buffer_provider.ptr;
            tokio::task::spawn_blocking(move || {
                // SAFETY: ptr is valid for the lifetime of the source.
                // getBufferBlocking is thread-safe (BufferManager uses atomics).
                let pin = unsafe { std::pin::Pin::new_unchecked(&mut *ptr) };
                let buf_ptr = nes_source_bindings::ffi::getBufferBlocking(pin);
                TupleBufferHandle::new(buf_ptr)
            })
            .await
            .expect("spawn_blocking task panicked in allocate_buffer")
        }

        /// Emit a filled buffer into the async channel.
        ///
        /// This will await if the channel is full (backpressure).
        /// Returns Ok(()) on success, or Err if the channel is closed
        /// (source is being shut down).
        pub async fn emit(&self, buffer: TupleBufferHandle) -> Result<(), SourceError> {
            self.sender
                .send(buffer)
                .await
                .map_err(|_| SourceError::new("emit channel closed -- source is shutting down"))
        }

        /// Get a clone of the cancellation token for cooperative shutdown.
        ///
        /// Source authors use this in `tokio::select!` to handle cancellation:
        /// ```ignore
        /// tokio::select! {
        ///     _ = ctx.cancellation_token().cancelled() => break,
        ///     // ... other branches
        /// }
        /// ```
        pub fn cancellation_token(&self) -> CancellationToken {
            self.cancellation_token.clone()
        }
    }
    ```

    IMPORTANT design note on `allocate_buffer`: The BufferProviderHandle stores a raw pointer. To send it across the spawn_blocking boundary, we extract the raw pointer (which is Copy) and reconstruct a temporary BufferProviderHandle in the blocking thread. Since BufferProviderHandle has no Drop behavior (it doesn't own the C++ object), `std::mem::forget` is just for clarity.

    Actually, simpler approach -- make BufferProviderHandle::ptr a pub(crate) field or add a method that returns the raw pointer, and let allocate_buffer use it directly:

    ```rust
    pub async fn allocate_buffer(&self) -> TupleBufferHandle {
        let ptr = self.buffer_provider.as_raw_ptr();
        tokio::task::spawn_blocking(move || {
            // SAFETY: ptr is valid for the source's lifetime.
            // getBufferBlocking is thread-safe (BufferManager uses atomics).
            let pin = unsafe { std::pin::Pin::new_unchecked(&mut *ptr) };
            let buf_ptr = nes_source_bindings::ffi::getBufferBlocking(pin);
            TupleBufferHandle::new(buf_ptr)
        })
        .await
        .expect("spawn_blocking task panicked in allocate_buffer")
    }
    ```

    Add to BufferProviderHandle:
    ```rust
    pub(crate) fn as_raw_ptr(&self) -> *mut ffi::AbstractBufferProvider {
        self.ptr
    }
    ```
