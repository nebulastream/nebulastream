pub struct BridgeMessage {
    pub origin_id: u64,
    pub buffer: TupleBufferHandle,
    pub permit: tokio::sync::OwnedSemaphorePermit,
}

pub struct BridgeHandle {
    pub sender: async_channel::Sender<BridgeMessage>,
    thread: Option<std::thread::JoinHandle<()>>,
}

pub type EmitFnPtr = unsafe extern "C" fn(*mut std::ffi::c_void, u64, *mut ffi::TupleBuffer) -> u8;

/// Per-source emit callback registry. Bridge loop looks up by origin_id.
/// Static EMIT_REGISTRY: DashMap<u64, EmitEntry>
pub struct EmitEntry {
    pub emit_fn: EmitFnPtr,
    pub emit_ctx: *mut std::ffi::c_void,
}

/// Registers a source's emit callback. MUST be called before spawning the source task.
pub fn register_emit(source_id: u64, emit_fn: EmitFnPtr, emit_ctx: *mut std::ffi::c_void);

/// Unregisters a source's emit callback. Called by monitoring task on source exit.
pub fn unregister_emit(source_id: u64);

/// Returns the shared channel sender. Takes NO emit callback parameters.
/// Emit dispatch is handled by EMIT_REGISTRY lookup in bridge_loop.
pub fn ensure_bridge() -> &'static async_channel::Sender<BridgeMessage>;

**1. Update `context.rs`:**

Add fields to SourceContext:
- `backpressure: Arc<BackpressureState>` -- from backpressure.rs registry
- `semaphore: Arc<tokio::sync::Semaphore>` -- per-source, capacity from inflight_limit
- `origin_id: u64` -- for BridgeMessage

Change `sender` field type from `Sender<TupleBufferHandle>` to `Sender<BridgeMessage>` (from bridge.rs).

Update `SourceContext::new()` to accept the new fields.

Rewrite `emit()` to follow the three-step pattern per user decision:
1. `self.backpressure.wait_for_release().await` -- yield if C++ backpressure applied
2. `self.semaphore.clone().acquire_owned().await` -- yield if inflight limit reached
3. `self.sender.send(BridgeMessage { origin_id, buffer, permit }).await` -- send to bridge

Keep `allocate_buffer()` and `cancellation_token()` unchanged.

**2. Create `source.rs`:**

Define `AsyncSource` trait:
```rust
pub trait AsyncSource: Send + 'static {
    fn run(
        &mut self,
        ctx: SourceContext,
    ) -> impl std::future::Future<Output = SourceResult> + Send;
}
```

Implement `spawn_source()` function:
- Takes: source_id (u64), source (impl AsyncSource), buffer_provider (*mut AbstractBufferProvider), inflight_limit (u32), emit_fn (EmitFnPtr), emit_ctx (*mut c_void), error_fn (fn pointer for on_source_error), error_ctx (*mut c_void)
- **CRITICAL: Register per-source emit callback BEFORE spawning the source task:**
  Call `bridge::register_emit(source_id, emit_fn, emit_ctx)` -- this ensures the bridge thread can dispatch messages for this source as soon as they arrive. Registration must happen before the source task can call emit().
- Creates: CancellationToken, Arc<Semaphore>(inflight_limit), registers backpressure state via register_source(source_id)
- Gets channel sender from `ensure_bridge().clone()` -- note: ensure_bridge takes NO emit callback args (dispatch handled by EMIT_REGISTRY)
- Constructs SourceContext with all fields
- Spawns source.run(ctx) as a Tokio task on the source runtime
- Spawns a monitoring task (tokio::spawn) that awaits the JoinHandle:
  - On Ok(SourceResult::Error(e)) or Err(panic): calls error_fn(error_ctx, source_id, c_string_msg)
  - On Ok(SourceResult::EndOfStream): logs info, no error callback
  - **In all cases: calls `unregister_source(source_id)` to clean up backpressure state AND `bridge::unregister_emit(source_id)` to remove emit callback from EMIT_REGISTRY.** Both cleanups happen in the monitoring task after the source task has exited, ensuring no more messages will be sent for this source. There is a small race window where the bridge may process a final in-flight message after unregister -- bridge_loop handles this by logging a warning for unregistered sources (not panicking).
- Returns Box<SourceTaskHandle>

**3. Create `handle.rs`:**

`SourceTaskHandle` struct containing:
- `cancellation_token: CancellationToken` -- for stop_source()
- `source_id: u64` -- for identification

Implement `stop_source(handle: &SourceTaskHandle)`:
- Calls `handle.cancellation_token.cancel()` -- non-blocking per user decision
- Logs that stop was requested

**4. Update `lib.rs`:**

Add `pub mod source;`, `pub mod handle;`. Re-export `AsyncSource`, `SourceTaskHandle`.

Note on error_fn: The error callback uses a C-style function pointer pattern identical to emit_fn. Type: `unsafe extern "C" fn(*mut c_void, u64, *const std::ffi::c_char)`. The monitoring task must create a CString from the error message before calling the function pointer.
