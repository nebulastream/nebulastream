```rust
extern "Rust" {
    type SourceTaskHandle;
    fn spawn_source(
        source_id: u64,
        buffer_provider: *mut AbstractBufferProvider,
        inflight_limit: u32,
        emit_fn: ...,     // C-style function pointer
        emit_ctx: ...,    // void* context
        error_fn: ...,    // error callback function pointer
        error_ctx: ...,   // void* context
    ) -> Result<Box<SourceTaskHandle>>;
    fn stop_source(handle: &SourceTaskHandle);
    fn on_backpressure_applied(source_id: u64);
    fn on_backpressure_released(source_id: u64);
}
```

Note on emit dispatch: spawn_source internally registers the emit_fn/emit_ctx pair in the
bridge's EMIT_REGISTRY keyed by source_id. The bridge thread dispatches each message to
the correct per-source emit callback via registry lookup. The C++ side does NOT interact
with the registry directly -- it only passes the emit callback to spawn_source, which
handles registration. The monitoring task unregisters on source exit.

**1. Create `TokioSource.hpp`:**

```cpp
class TokioSource {
public:
    explicit TokioSource(OriginId originId, uint32_t inflightLimit);
    ~TokioSource();

    // Same interface as SourceThread
    [[nodiscard]] bool start(SourceReturnType::EmitFunction&& emitFunction);
    void stop();
    [[nodiscard]] SourceReturnType::TryStopResult tryStop(std::chrono::milliseconds timeout);
    [[nodiscard]] OriginId getSourceId() const;

private:
    OriginId originId;
    uint32_t inflightLimit;
    rust::Box<SourceTaskHandle> rustHandle;  // Opaque Rust handle (per user decision)
    std::unique_ptr<EmitContext> emitContext;
    std::unique_ptr<ErrorContext> errorContext;
    bool started{false};
};
```

**2. Create `TokioSource.cpp`:**

`start(EmitFunction&& emitFn)`:
- Create EmitContext with emitFn and a new stop_source
- Create ErrorContext with sourceId
- Call Rust FFI `spawn_source(originId, bufferProvider, inflightLimit, bridge_emit, emitContext.get(), on_source_error_callback, errorContext.get())`
  - Note: spawn_source internally registers bridge_emit/emitContext in EMIT_REGISTRY keyed by originId. The C++ side does not need to interact with the registry.
- Store returned Box<SourceTaskHandle> in rustHandle
- Set started=true
- Return true on success, catch exceptions and return false

`stop()`:
- If started: call Rust FFI `stop_source(*rustHandle)` -- non-blocking per user decision
- Also request_stop on the stop_source (for the C++ side stop_token)

`tryStop(timeout)`:
- Call stop()
- Return TryStopResult based on whether the source has already exited (can check via a flag set by the error callback, or simply return StopRequested since async sources stop asynchronously)

`~TokioSource()`:
- If started and not stopped: call stop()
- rustHandle is dropped by Rust Box destructor
