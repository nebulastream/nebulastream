<objective>
Extract validation code (ConfigParameters structs and RegisterXxxValidation functions) from all source and sink types into dedicated validation files. Create nes-sources-validation and nes-sinks-validation CMake targets that link only nes-common + nes-configurations. Wire runtime targets (nes-sources, nes-sinks) to link against validation targets.

Purpose: Separate validation-only code from runtime dependencies so WASM validator can link real config validation instead of pass-through stubs.
Output: Two new CMake targets (nes-sources-validation, nes-sinks-validation) with all source/sink validation code extracted, all existing tests passing.
</objective>

```cmake
add_subdirectory(src)

# --- Validation-only target (no runtime deps) ---
add_library(nes-sources-validation
    src/FileSourceValidation.cpp
    src/SourceValidationProvider.cpp
)
target_link_libraries(nes-sources-validation PUBLIC nes-common nes-configurations)
target_include_directories(nes-sources-validation PUBLIC
    $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include>
    $<INSTALL_INTERFACE:include/nebulastream/>
)

# Create validation registry (uses existing registry/templates/ in nes-sources/)
create_plugin_registry_library(nes-sources-validation-registry nes-sources-validation)
target_link_libraries(nes-sources-validation PRIVATE nes-sources-validation-registry)
generate_plugin_registrars(nes-sources-validation SourceValidation)

# --- Runtime target ---
get_source(nes-sources NES_SOURCES_SOURCE_FILES)
add_library(nes-sources ${NES_SOURCES_SOURCE_FILES})
target_link_libraries(nes-sources PUBLIC nes-sources-validation nes-common nes-configurations nes-memory nes-executable)
target_include_directories(nes-sources PUBLIC
    $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include>
    $<INSTALL_INTERFACE:include/nebulastream/>
    PRIVATE $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/private>
)
if (NES_ENABLE_PRECOMPILED_HEADERS)
    target_precompile_headers(nes-sources REUSE_FROM nes-common)
endif()

# Runtime registries (Source, FileData, InlineData -- NOT SourceValidation, owned by validation target)
create_registries_for_component(Source FileData InlineData)
add_tests_if_enabled(tests)
```

CRITICAL: The `add_plugin_as_library` calls in plugin CMakeLists.txt files currently use `nes-sources-registry` as the third parameter for SourceValidation plugins. After the split, the SourceValidation plugins need to link against `nes-sources-validation-registry` instead. Update ALL plugin CMakeLists.txt:
- TCPSource: `add_plugin_as_library(TCP SourceValidation nes-sources-validation-registry tcp_source_validation_plugin_library TCPSourceValidation.cpp)`
- GeneratorSource: `add_plugin_as_library(Generator SourceValidation nes-sources-validation-registry generator_validation_plugin_library GeneratorSourceValidation.cpp Generator.cpp GeneratorFields.cpp FixedGeneratorRate.cpp SinusGeneratorRate.cpp)`
- NetworkSource: `add_plugin_as_library(Network SourceValidation nes-sources-validation-registry network_source_validation_plugin_library NetworkSourceValidation.cpp)`

Note: The `add_plugin_as_library` function stores `plugin_name` and `plugin_library` in global properties keyed by `plugin_registry` ("SourceValidation"). Then `generate_plugin_registrars` (called for `nes-sources-validation`) iterates these global properties and links plugin libraries against `nes-sources-validation`. This wiring should work as-is since `generate_plugin_registrar` links plugin libraries against the `plugin_registry_component` parameter.
