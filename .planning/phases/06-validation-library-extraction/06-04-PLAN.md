  <action>
    1. Study `nes-frontend/apps/cli/CLIStarter.cpp` to identify the offline validation path. The CLI has two modes:
       - Online: submits queries to a running NES coordinator (via gRPC)
       - Offline: validates topology YAML locally (the path we're replacing)

       Find the offline validation code path — it will be the section that:
       - Reads a YAML file
       - Populates SourceCatalog/SinkCatalog from YAML
       - Validates queries via StatementBinder
       - Reports errors

    2. Add nes-validator as a dependency:
       - In `nes-frontend/CMakeLists.txt`: add `nes-validator` to `target_link_libraries(nes-frontend-lib ...)` (PUBLIC or PRIVATE — PRIVATE is preferred since it's an implementation detail)
       - Alternatively, if only nes-cli needs it (not nes-repl), add it in `nes-frontend/apps/CMakeLists.txt` to the nes-cli target directly: `target_link_libraries(nes-cli PUBLIC nes-frontend-lib argparse::argparse yaml-cpp::yaml-cpp nes-validator)`

    3. In CLIStarter.cpp, add `#include <Validator/TopologyValidator.hpp>` and replace the offline validation code path with a call to `NES::Validator::validateTopology(yamlString)`:
       - Read the YAML file content into a string (this part stays in CLIStarter)
       - Call `auto error = NES::Validator::validateTopology(yamlString);`
       - If error is empty: report success
       - If error is non-empty: report the error message and exit with failure

       IMPORTANT: Do NOT remove the online submission path (gRPC query submission). Only replace the local validation logic. The CLI's `--validate` or `--offline` flag (or equivalent) should use nes-validator; the online submit path remains untouched.

       If CLIStarter.cpp has the YAML parsing and validation inline (not factored into separate functions), you may need to carefully identify which code block to replace. Look for the try-catch block that creates SourceCatalog, calls loadStatements(), and runs StatementBinder — that entire block should be replaced with the single validateTopology() call.

    4. Keep the removed inline code as a comment block or in a separate deprecated file for reference during review. This makes it easy to verify the extraction is complete and correct.

    5. Build nes-cli to verify it compiles:
       ```
       cmake --build cmake-build-debug --target nes-cli -j$(nproc)
       ```

    6. If the offline bats tests exist and can run without a NES cluster, run them:
       ```
       ctest --test-dir cmake-build-debug -R "offline-cli-test" --output-on-failure
       ```
       If they require additional setup or a running NES instance, document what's needed but don't block on it.
  </action>
