From research — locked decisions for schema tests:
- Construct test Model objects via deserializeModel() from SerializableOperator_Model protobuf
- Use mixed data types: Float32, Int32, Int64, and VARSIZED
- VARSIZED type must be accepted by the operator
- Error paths throw CannotInferSchema — test with EXPECT_THROW
- Name collision (SCHEMA-07): output fields with same name get type replaced, not error

From research — Schema inference logic ported into withInferredSchema:
1. Check inputSchemas not empty → CannotInferSchema
2. Check inputFieldNames.size() matches model.getInputs().size() → CannotInferSchema
3. For each inputFieldName, check field exists in schema → CannotInferSchema
4. For each inputFieldName, check type matches model input type → CannotInferSchema
5. Append output fields (replace on name collision)

From research — BaseUnitTest macros:
