Add a destructor to `IREERuntimeWrapper` that safely releases the IREE session and instance handles. CRITICAL details from research:

1. **In the header** (`IREERuntimeWrapper.hpp`):
   - Add `~IREERuntimeWrapper();` declaration
   - Add rule-of-five: delete copy constructor and copy assignment, default move constructor and move assignment (moves are safe since the raw pointers transfer ownership; the moved-from object's pointers become indeterminate but won't double-free if set to nullptr in move)
   - Actually, simplest safe approach: delete copy AND move (non-movable, non-copyable). The wrapper is used as a member of `InferModelPhysicalOperator` which itself is non-copyable.

2. **In the implementation** (`IREERuntimeWrapper.cpp`):
   - Add destructor that releases in order: session first, then instance
   - Do NOT release `device` — `setup()` already calls `iree_hal_device_release(device)` inline after session creation, so the stored `device` pointer is dangling. Releasing it again would double-free.
   - Guard each release with a null check (the pointers are default-initialized to garbage in the default constructor, so also initialize them to `nullptr` in the header or constructor)

3. **Initialize raw pointers to nullptr** in the header's member declarations:
   ```cpp
   iree_runtime_instance_t* instance = nullptr;
   iree_runtime_session_t* session = nullptr;
   iree_hal_device_t* device = nullptr;
   ```

4. The `iree_vm_function_t function` member is a value type (struct), no release needed.

Destructor implementation:
```cpp
IREERuntimeWrapper::~IREERuntimeWrapper() {
    if (session) {
        iree_runtime_session_release(session);
    }
    if (instance) {
        iree_runtime_instance_release(instance);
    }
}
```
