Header (`InferModelLogicalOperator.hpp`):
- `#include <Model.hpp>` from nes-inference
- Constructor: `InferModelLogicalOperator(Nebuli::Inference::Model model, std::vector<std::string> inputFieldNames)`
- Getter: `[[nodiscard]] Nebuli::Inference::Model getModel() const;`
- Getter: `[[nodiscard]] std::vector<std::string> getInputFieldNames() const;`
- All LogicalOperatorConcept methods (same signatures as Selection but returning InferModelLogicalOperator)
- `static constexpr std::string_view NAME = "InferModel";`
- Private members: `Nebuli::Inference::Model model`, `std::vector<std::string> inputFieldNames`, `std::vector<LogicalOperator> children`, `TraitSet traitSet`, `Schema inputSchema, outputSchema`
- `Reflector<InferModelLogicalOperator>` and `Unreflector<InferModelLogicalOperator>` specializations
- `static_assert(LogicalOperatorConcept<InferModelLogicalOperator>)`
- `detail::ReflectedInferModelLogicalOperator` struct for reflection

Implementation (`InferModelLogicalOperator.cpp`):
- Follow SelectionLogicalOperator.cpp pattern exactly for all concept methods
- `explain()`: return `fmt::format("INFER_MODEL({})", model.getName())` for normal verbosity, add opId and traitSet for Debug verbosity. If `model.getName()` doesn't exist, use the model's output field names or a generic description.
- `operator==`: compare model and inputFieldNames (and outputSchema, inputSchemas, traitSet like Selection does)
- `withInferredSchema()`: This is where the schema inference logic from ls-inference gets ported. Implement per the research Architecture Patterns section — validate input count, check type compatibility, append/replace output fields. Use `CannotInferSchema` exception for errors. Use immutable Schema API (`schema = schema.addField(...)` reassignment pattern). Check if `Schema::replaceField` exists; if not, remove-then-add. VARSIZED type must be accepted (no special rejection).
- `Reflector`/`Unreflector`: Follow Selection pattern. Reflect model and inputFieldNames.
- `LogicalOperatorGeneratedRegistrar::RegisterInferModelLogicalOperator`: Follow Selection's Register pattern.
