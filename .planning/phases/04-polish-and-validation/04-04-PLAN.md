    - Test 1: toggleDarkMode flips darkMode state in store
    - Test 2: toggleDarkMode adds/removes 'dark' class on document.documentElement
    - Test 3: toggleDarkMode persists 'theme' key in localStorage
  </behavior>
  <action>
    1. Update `tailwind.config.js`: add `darkMode: 'class'` to the config object.

    2. Add to `uiSlice.ts`:
       - `darkMode: boolean` state (initialized from `document.documentElement.classList.contains('dark')` or just `false` -- the inline script in index.html handles initial state)
       - `toggleDarkMode: () => void` action that:
         a. Flips `darkMode` in state
         b. Calls `document.documentElement.classList.toggle('dark')`
         c. Calls `localStorage.setItem('theme', newValue ? 'dark' : 'light')`

    3. Add inline script to `index.html` `<head>` section (BEFORE any CSS/JS loads) to prevent flash:
       ```html
       <script>
         (function() {
           var t = localStorage.getItem('theme');
           if (t === 'dark' || (!t && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
             document.documentElement.classList.add('dark');
           }
         })();
       </script>
       ```

    4. In `Toolbar.tsx`: add a toggle button with `Sun` icon (light mode) or `Moon` icon (dark mode) from lucide-react. On click, call `toggleDarkMode()`. Read `darkMode` from store to determine which icon to show.

    5. Write tests in `src/__tests__/dark-mode.test.ts`. These test the store action directly -- call toggleDarkMode, assert state flips and document.documentElement.classList is updated. Use jsdom environment.
  </action>

  </verify>
