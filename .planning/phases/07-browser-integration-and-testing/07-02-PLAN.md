  <action>
1. Create `nes-topology-editor/src/hooks/useWasmValidation.ts`:
   - Creates the Web Worker on mount using `new Worker(new URL('../workers/validationWorker.ts', import.meta.url), { type: 'module' })`
   - Stores worker in a ref, terminates on unmount cleanup
   - Handles worker messages:
     - `ready`: set wasmStatus to 'ready', resetRetryCount, trigger initial validation immediately
     - `result`: if result.id matches latest requestIdRef, set semanticError to result.error, set wasmStatus to 'ready'
     - `error`: incrementRetryCount, if retryCount < MAX_RETRIES then re-create the worker and retry, else set wasmStatus to 'error' and setWasmError(message)
   - Subscribes to topology-relevant store state: workers, logicalSources, physicalSources, sinks, queries
   - On state change (useEffect dependency array): if wasmStatus is 'ready', clear previous debounce timer, set new timer (300ms), on fire: generate YAML via storeToYaml(), increment requestIdRef, postMessage({type: 'validate', id, yaml}), set wasmStatus to 'validating'
   - Stale result protection: only apply result if response.id matches the latest requestIdRef.current
   - Export as default hook: `export function useWasmValidation(): void`

2. Create `nes-topology-editor/src/components/StatusBar/StatusBar.tsx`:
   - Positioned at bottom of canvas-area (absolute bottom-left, z-index above canvas)
   - Uses lucide-react icons: CheckCircle2, AlertTriangle, Loader2, XCircle
   - Reads from store: wasmStatus, semanticError, wasmError
   - States:
     - wasmStatus === 'loading': Loader2 (animate-spin) + "Loading validator..." text (muted)
     - wasmStatus === 'validating': Loader2 (animate-spin) + "Validating..." text (muted)
     - wasmStatus === 'ready' && semanticError === null: CheckCircle2 (green) — not yet validated (brief)
     - wasmStatus === 'ready' && semanticError === '': CheckCircle2 (green) + "Valid" text
     - wasmStatus === 'ready' && semanticError is non-empty: AlertTriangle (amber) with hover tooltip showing the error message (title attribute or a small popover)
     - wasmStatus === 'error': XCircle (red) + "Validator unavailable" text, title shows wasmError
   - Tailwind classes for styling: small pill/bar, semi-transparent background, rounded, padding
   - Respect darkMode from store for appropriate colors

3. Update `nes-topology-editor/src/components/YamlOverlay/YamlOverlay.tsx`:
   - Below the existing Monaco editor, add a conditional section that shows when semanticError is non-empty
   - Read semanticError from store: `const semanticError = useStore(s => s.semanticError);`
   - Display: a div below the editor with a warning icon and the error text, styled with amber/red border-left, small font, padding
   - Only visible when yamlOverlayVisible is true AND semanticError is a non-empty string
   - Style to look like a "Problems" panel — monospace font, compact

4. Update `nes-topology-editor/src/App.tsx`:
   - Import and call `useWasmValidation()` hook inside the App component (before the return)

   - Add `<StatusBar />` inside the `canvas-area` div (after `<YamlOverlay />`)
