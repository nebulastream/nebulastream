  <behavior>
    - addWorker() creates a worker with auto-generated host "worker-N:9090" and unique ID
    - removeWorker(id) removes a worker and its associated edges
    - connectDownstream(sourceWorkerId, targetWorkerId) creates a downstream link
    - attachSourceToWorker(sourceId, workerId) updates the source's host worker reference
    - attachSinkToWorker(sinkId, workerId) updates the sink's host worker reference
    - updateNodePosition(id, position) updates position for any node type
    - selectReactFlowNodes returns Node[] with correct types ('worker', 'source', 'sink') and data
    - selectReactFlowEdges returns Edge[] including downstream edges and source/sink-to-worker placement edges
    - Workers are type 'worker' with large dimensions, sources are type 'source', sinks are type 'sink'
  </behavior>
  <action>
    1. Create `src/lib/types.ts` with domain types mapping to NES YAML format:
       - `Worker`: id, host (e.g. "worker-1:9090"), grpc (e.g. "worker-1:8080"), capacity (default 10000), downstream (string[] of worker IDs), position ({x, y})
       - `LogicalSource`: id, name, schema ({name, type}[])
       - `PhysicalSource`: id, logicalSourceId, hostWorkerId, type ('Generator'|'CSV'|etc), sourceConfig (Record<string,string>), parserConfig (Record<string,string>), position ({x, y})
       - `Sink`: id, name, hostWorkerId, type ('File'|'Void'|'Print'|etc), schema ({name, type}[]), config (Record<string,string>), position ({x, y})
       - `TopologyState`: workers[], logicalSources[], physicalSources[], sinks[]

    2. Create `src/lib/ids.ts` with `generateId()` using `crypto.randomUUID()` and `generateWorkerHost(existingWorkers)` that returns "worker-N:9090" where N is next available number.

    3. Create `src/store/topologySlice.ts`:
       - State: workers, logicalSources, physicalSources, sinks
       - Actions: addWorker(position?), removeWorker(id), addPhysicalSource(source), removePhysicalSource(id), addSink(sink), removeSink(id), connectDownstream(fromId, toId), disconnectDownstream(fromId, toId), attachSourceToWorker(sourceId, workerId), attachSinkToWorker(sinkId, workerId), updateNodePosition(id, position)

    4. Create `src/store/uiSlice.ts`:
       - State: selectedNodeId (string|null), clipboard (serialized nodes/edges for copy/paste)
       - Actions: selectNode(id|null), setClipboard(data)

    5. Create `src/store/index.ts` combining slices into a single Zustand store using `create()`. Export `useStore`.

    6. Create `src/store/selectors.ts`:
       - `selectReactFlowNodes(state)`: Maps workers to Node type 'worker' (data: {worker}), physicalSources to type 'source' (data: {source, color}), sinks to type 'sink' (data: {sink}). Color for sources derived from logicalSourceId (use a stable color palette indexed by logical source).
       - `selectReactFlowEdges(state)`: Creates edges for (a) worker downstream connections, (b) source-to-worker placement edges, (c) sink-to-worker placement edges.

    7. Write tests in `src/__tests__/store.test.ts` covering:
       - Worker CRUD and auto-naming
       - Downstream connection management
       - Source/sink attachment and reassignment
       - Position updates
       - Selector output shape (nodes have correct type, edges have correct source/target)
  </action>
