       - Call `isValidConnectionType(sourceNodeType, targetNodeType)` -- reject if false

    2. Add `onConnect` callback to Canvas.tsx:
       - Determine connection type from source/target node types
       - If worker-to-worker: call `store.connectDownstream(connection.source, connection.target)`
       - If source-to-worker: call `store.attachSourceToWorker(connection.source, connection.target)`
       - If sink-to-worker: call `store.attachSinkToWorker(connection.source, connection.target)`

    3. Add invalid connection feedback:
       - When `isValidConnection` returns false, show a brief toast/message. Simplest approach: use a state variable to show a fading message near the cursor or at top of canvas. Message text: "Cannot connect: would create a cycle" for cycle violations, "Invalid connection type" for type violations.
       - Alternative: use `connectionLineStyle` to make the line red when hovering an invalid target (React Flow supports this via CSS on `.react-flow__connection-line` when connection is invalid).

    4. Add `onDelete` handler to Canvas.tsx:
       - React Flow fires this when user presses Backspace/Delete with selected nodes/edges
       - For each deleted node: call the appropriate store removal (removeWorker, removePhysicalSource, or removeSink based on node type)
       - For each deleted edge: call store to disconnect (disconnectDownstream or detach source/sink)
       - Ensure the `deleteKeyCode` prop is set to `["Backspace", "Delete"]` for both keys

    5. If needed, add any missing store actions to `topologySlice.ts`:
       - `removeEdge(edgeId)` or equivalent for removing downstream connections and source/sink attachments by edge
       - Ensure `removeWorker` also cleans up: removes the worker from other workers' downstream arrays, detaches any sources/sinks that referenced this worker
