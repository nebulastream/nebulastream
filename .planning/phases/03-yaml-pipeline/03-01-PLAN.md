<feature>
  <name>YAML Serialization/Deserialization</name>
  <files>nes-topology-editor/src/lib/yaml.ts, nes-topology-editor/src/lib/yaml.test.ts</files>
  <behavior>
    **storeToYaml:**
    - Empty topology -> empty YAML (just `{}` or empty string)
    - Single worker -> `workers: [{host, grpc, capacity}]` (no downstream key when empty)
    - Worker with downstream -> `workers: [{host, grpc, capacity, downstream: [host-string]}]` (UUID resolved to host)
    - Queries -> `queries: [{query: sql, name: name}]` (future format per user decision)
    - Sinks -> `sinks: [{name, host, schema: [{name, type}], type, config}]` (hostWorkerId UUID resolved to host string)
    - Logical sources -> `logical: [{name, schema: [{name, type}]}]`
    - Physical sources -> `physical: [{logical: name-string, host: host-string, type, parser_config, source_config}]` (logicalSourceId UUID -> name, hostWorkerId UUID -> host)
    - Key ordering: queries, sinks, logical, physical, workers
    - Omit empty sections (no `workers: []`)

    **yamlToStore:**
    - Parse workers, create UUIDs, build host->UUID map
    - Resolve downstream host references to UUIDs
    - Parse logical sources, build name->UUID map
    - Parse physical sources, resolve logical name and host to UUIDs
    - Parse sinks, resolve host to UUIDs
    - Parse queries -- accept BOTH old format (`query: string|string[]`) and new format (`queries: [{query, name}]`)
    - All returned entities have generated UUIDs and position {x:0, y:0}
    - Invalid YAML string throws YAMLException (from js-yaml)

    **Roundtrip:**
    - Given a full topology, yamlToStore(storeToYaml(topology)) produces equivalent data (UUIDs differ, but names/hosts/structure match)
  </behavior>
  <implementation>
    Create `nes-topology-editor/src/lib/yaml.ts` with:
    1. `storeToYaml(workers, logicalSources, physicalSources, sinks, queries): string` -- pure function, uses `yaml.dump()` from js-yaml with `{lineWidth: -1, noRefs: true}`. Build worker ID->host map and logical ID->name map for reference resolution. Omit empty sections.
    2. `yamlToStore(yamlStr: string): ParseResult` -- pure function, uses `yaml.load()`. Generate UUIDs via `generateId()`, build host->UUID and name->UUID maps, resolve cross-references. Support both query formats.

    Follow the code examples in 03-RESEARCH.md closely. Use `import yaml from 'js-yaml'` and `import { generateId } from './ids'`.
  </implementation>
</feature>
