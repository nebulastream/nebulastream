  <action>
    **autoLayout.ts:** Create a standalone helper function:
    ```typescript
    import { useStore } from '../store';
    import { selectReactFlowNodes, selectReactFlowEdges } from '../store/selectors';
    import { getLayoutedElements } from './layout';

    export function applyAutoLayout(): void {
      const state = useStore.getState();
      const nodes = selectReactFlowNodes(state);
      const edges = selectReactFlowEdges(state);
      if (nodes.length === 0) return;
      const { nodes: layouted } = getLayoutedElements(nodes, edges, 'LR');
      state.setAllNodePositions(layouted.map(n => ({ id: n.id, position: n.position })));
    }
    ```
    This reads store state directly (no hooks), computes layout, writes positions back. Direction is 'LR' (left-to-right) matching existing Toolbar pattern.

    **Canvas.tsx modifications:**
    1. Import `applyAutoLayout` from `../../lib/autoLayout`.
    2. **Remove free positioning from onNodeDragStop:** Currently the handler calls `updateNodePosition(node.id, ...)` on every drag stop. Change this:
       - If no drop target found (plain drag, not drag-to-connect/reassign), do NOT update position. Instead, the node snaps back to its auto-layouted position on next render (store positions unchanged).
       - If a drop target IS found and a structural change happens (connectDownstream, attachSourceToWorker, attachSinkToWorker), call `applyAutoLayout()` AFTER the store action.
    3. **Auto-layout on structural changes:** Add `applyAutoLayout()` calls after these existing store operations in Canvas:
       - After `connectDownstream` in onNodeDragStop and onConnect
       - After `attachSourceToWorker` and `attachSinkToWorker` in onNodeDragStop and onConnect
       - After `onDelete` (node/edge deletion changes graph structure)
       - After `addNodes` (paste operation)
    4. **Auto-layout on initial mount and topology changes:** Add a `useEffect` that calls `applyAutoLayout()` when the structural identity of the topology changes. Use a memo'd "structural fingerprint" (e.g., `workers.length + '-' + physicalSources.length + '-' + sinks.length + '-' + workers.map(w => w.downstream.join(',')).join(';')`) to detect structural changes vs. just position changes. This covers the case where `replaceTopology` is called from YAML import (Plan 02).
    5. **Remove the Toolbar auto-layout button:** Since layout is always automatic, the manual "Auto Layout" button in Toolbar.tsx is no longer needed. Remove the button and its handler. Keep the "Add Worker" button. Also call `applyAutoLayout()` after `addWorker` in the Toolbar's handleAddWorker.

    **Key subtlety:** When onNodeDragStop fires without a structural change (user just dragged a node around), we must NOT call updateNodePosition. The node will visually snap back to its layouted position because the store positions are unchanged and the next render cycle restores them from store. This is the desired behavior per user decision.
