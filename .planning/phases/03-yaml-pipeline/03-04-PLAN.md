    **2. Live preview (OUTP-01):**
    - Open the YAML overlay
    - Add a worker via the toolbar "Add Worker" button
    - Verify YAML updates immediately to show the new worker
    - Click the worker and edit its host in the properties panel
    - Verify YAML updates to reflect the new host value

    **3. NES format (OUTP-02):**
    - With workers, sources, sinks, and queries added, verify the YAML has correct structure:
      - `queries:` with `[{query: ..., name: ...}]` format
      - `sinks:` with host resolved (not UUID)
      - `logical:` with schema
      - `physical:` with logical name (not UUID) and host (not UUID)
      - `workers:` with downstream as host addresses (not UUIDs)

    **4. Export via copy (OUTP-03):**
    - In the YAML overlay, select all (Ctrl+A) and copy (Ctrl+C)
    - Paste into a text editor -- verify valid YAML

    **5. Import via paste (OUTP-04):**
    - Clear the editor (select all, delete)
    - Paste this test YAML:
    ```yaml
    queries:
      - query: "SELECT * FROM sensor_data INTO output_sink"
        name: "test_query"
    workers:
      - host: "w1:9090"
        grpc: "w1:8080"
        capacity: 5000
        downstream:
          - "w2:9090"
      - host: "w2:9090"
        grpc: "w2:8080"
        capacity: 10000
    logical:
      - name: sensor_data
        schema:
          - name: temperature
            type: FLOAT64
    physical:
      - logical: sensor_data
        host: "w1:9090"
        type: Generator
        source_config:
          generator_rate_type: FIXED
    sinks:
      - name: output_sink
        host: "w2:9090"
        type: Void
        schema:
          - name: temperature
            type: FLOAT64
        config: {}
    ```
    - Verify after ~500ms: canvas shows 2 workers connected, 1 source on w1, 1 sink on w2
    - Verify nodes are auto-layouted (not at position 0,0)

    **6. Error handling:**
    - Type invalid YAML (e.g., missing colon) in the overlay
    - Verify red squiggly underline appears at error location
    - Verify canvas does NOT change (stays at last valid state)
    - Fix the YAML -- verify error markers disappear and canvas updates

    **7. Always auto-layout:**
    - Drag a worker node to a different position
    - Release -- verify it snaps back to its auto-layouted position
    - Drag a source onto a different worker -- verify it reassigns AND auto-layouts
    - Add a new worker -- verify all nodes reposition via auto-layout
