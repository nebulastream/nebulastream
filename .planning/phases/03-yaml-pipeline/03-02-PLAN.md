    **YamlOverlay.tsx:** Create floating overlay component using inline styles (consistent with existing pattern):
    - Position: absolute, right: 16px, top: 16px, width ~500px, height ~70vh, z-index 500
    - Contains Monaco editor with `defaultLanguage="yaml"`, light theme
    - Monaco config: minimap disabled, word wrap on, scrollBeyondLastLine false
    - Bidirectional sync with echo prevention:
      1. Use `useRef(false)` as `isFromYamlEdit` flag
      2. Store->YAML direction: `useEffect` watches workers, logicalSources, physicalSources, sinks, queries from useStore. When they change AND `isFromYamlEdit.current` is false, call `storeToYaml()` and update the editor value via `editorRef.current.getModel().setValue(newYaml)`. When `isFromYamlEdit.current` is true, reset it to false and skip.
      3. YAML->Store direction: Monaco `onChange` handler debounces ~500ms (use useRef for setTimeout). On debounce fire: try `yamlToStore(value)`, if success set `isFromYamlEdit.current = true` then call `replaceTopology(parsed)` and `replaceQueries(parsed.queries)`. If parse fails, set Monaco error markers via `monaco.editor.setModelMarkers()` with the error line/column from the YAMLException.
      4. On successful parse, clear error markers.
    - Use `onMount` callback to capture both the editor instance and the monaco namespace for marker API access.
    - Show a small header bar with "YAML" title and a close (X) button that calls `toggleYamlOverlay`.
    - Render conditionally based on `yamlOverlayVisible` from store.

    **CRITICAL pitfall avoidance:**
    - Do NOT use Monaco's `value` prop for updates after mount -- it resets cursor position. Use `editor.getModel().setValue()` only when change comes from store (not user typing).
    - The echo prevention ref is essential: without it, store update -> YAML regen -> onChange -> parse -> store update = infinite loop.

    **Toolbar.tsx:** Add a "YAML" toggle button (use `Code` icon from lucide-react). On click, call `toggleYamlOverlay`. Style consistently with existing toolbar buttons. Show active state when overlay is visible (e.g., different background color).

    **App.tsx:** Import and render `<YamlOverlay />` inside the `canvas-area` div (after `<Canvas />`), so it floats over the canvas. It renders conditionally based on store state internally.
